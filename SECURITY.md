# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it
**privately** so maintainers can investigate and resolve the issue before it is
publicly disclosed.

### How to Report

1. Navigate to the main page of the GitHub repository.
2. Click the **Security** tab.
3. Select **Report a vulnerability**.
4. Complete the advisory form with as much detail as possible.

### What to Include

When submitting a report, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Proof of concept code, screenshots, or relevant logs (if available)
- The potential impact or severity
- Suggested fixes or mitigation ideas (if available)

For more information, refer to [Privately reporting a security vulnerability](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)

---

## Responsible Disclosure

Please follow responsible disclosure practices:

- Report vulnerabilities **privately** through GitHub Security Advisories.
- Allow maintainers reasonable time to investigate and address the issue.
- Avoid publicly disclosing the vulnerability until a fix or mitigation is available.

---

## Guidelines for Reporting

### Do

- Communicate clearly and respectfully.
- Provide detailed and reproducible steps whenever possible.
- Include supporting materials such as logs, stack traces, or proof of concept code.
- Review existing advisories to avoid submitting duplicate reports.

### Do Not

- Publicly disclose vulnerabilities before maintainers have had an opportunity to investigate and resolve the issue.
- Exploit vulnerabilities beyond what is necessary to demonstrate their existence.
- Submit fixes or modify repository code related to a vulnerability without first coordinating with the maintainers.

---

## Maintainer Response

Maintainers will make a best effort to:

- Acknowledge receipt of the report within a reasonable timeframe.
- Investigate and validate the reported vulnerability.
- Provide updates on progress when appropriate.
- Release a fix or mitigation if the vulnerability is confirmed.

Response times may vary depending on maintainer availability.

---

## Disclosure Timeline

If no meaningful progress or communication has occurred within **90 days**
of the initial report, the reporter may choose to publicly disclose the
vulnerability as part of responsible disclosure practices.

---

We appreciate the efforts of contributors who help improve the security and reliability of this project.
